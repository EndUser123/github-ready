🔄 Phase Breakdown

Phase 1

Diagnose & Prep

Clear interference sources before building

  • Check existing structure
  • Clear state files
  • Check for existing modules
Phase 1.5

Detect Package Type

Intelligent package type detection

  • Claude Code Plugin (.claude-plugin/)
  • Claude Skill (SKILL.md)
  • Python Library (src/)
  • Brownfield conversion option
Phase 2

Build Structure

Create appropriate directory structure

  • Plugin: core/, hooks/, commands/
  • Skill: skill/, resources/
  • Library: src/, tests/, pyproject.toml
Phase 3

Generate Templates

Create README, LICENSE, config files

  • README.md with badges
  • Three deployment models
  • LICENSE (MIT default)
  • Configuration files
Phase 4

Validate

Verify package structure is correct

  • Platform compatibility check
  • Symlink test
  • Pytest collect
  • Tree diff
Phase 4.5

Code Review

Automated quality validation

  • Security issues
  • Performance problems
  • Maintainability concerns
  • Cross-file analysis
Phase 4.7

Media Generation

Create professional portfolio assets

  • Review bundle generation
  • NotebookLM videos/diagrams
  • C4 Mermaid diagrams
  • Interactive HTML diagrams
Phase 5

Portfolio Polish

Transform into portfolio-quality artifact

  • Badges (coverage, version)
  • CI/CD workflows
  • Documentation
  • CHANGELOG.md
Phase 6

Cleanup

Remove obsolete files

  • Backup files
  • Orphaned tests
  • Obsolete docs
Phase 7

Git Ready

Initialize git repository

  • git init
  • Initial commit
  • Set main branch

🚀 Quick Start

Create New Package

  • /package mylib
  • Auto-detects package type
  • Full pipeline execution

Skip Media Generation

  • /package mylib --skip media
  • Faster iteration
  • Skip NotebookLM assets

Preview Mode

  • /package --dry-run
  • See what will be created
  • No file writes
