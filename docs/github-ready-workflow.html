A step-by-step view of the package creation pipeline, from package-type detection through validation, media generation, cleanup, and git-ready output.

Pipeline overview

The workflow favors deterministic packaging steps first, then quality gates, then media and presentation polish.

Phase breakdown

Phase 1: Diagnose and prep

  • Inspect current structure
  • Clear local state artifacts
  • Check for existing modules

Phase 1.5: Detect package type

  • Claude Code plugin
  • Claude skill
  • Python library
  • Brownfield conversion path

Phase 2: Build structure

  • Create canonical directories
  • Write config and metadata
  • Prepare tests and docs layout

Phase 3: Generate templates

  • README and docs
  • License and changelog
  • Package-specific boilerplate

Phase 4: Validate

  • Platform compatibility checks
  • Pytest collect
  • Tree diff and sanity checks

Phase 4.5: Code review

  • Security and performance
  • Maintainability review
  • Cross-file analysis

Phase 4.7: Media generation

  • NotebookLM exports
  • Overview image
  • Flowchart + HTML pages
  • Explainer video

Phase 5 to 7

  • Portfolio polish
  • Cleanup stale files
  • Prepare git-ready output
